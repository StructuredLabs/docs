Structured’s Knowledge Base is a comprehensive repository that contains all the entities, metrics, and data definitions relevant to your organization. By providing a centralized location for this information, the Knowledge Base ensures that every team member has access to consistent, accurate, and up-to-date data, fostering better decision-making and collaboration across departments.

## **What is the Knowledge Base in Structured?**

The Knowledge Base in Structured is a centralized library where all standardized business metrics, entities, and data definitions are stored. It acts as a single source of truth for your organization, ensuring that every department works with the same definitions and metrics, thereby eliminating discrepancies and miscommunication.

### **Key Features of the Knowledge Base**

<AccordionGroup>
  <Accordion title="Centralized Data Repository" icon="database">
    The Knowledge Base serves as a centralized repository where all relevant
    business metrics, data entities, and definitions are stored. This
    centralization ensures that everyone in the organization can easily access
    the information they need, reducing the risk of using outdated or incorrect
    data.
  </Accordion>
  <Accordion title="Integration with Semantic Layer" icon="layer-group">
    The Knowledge Base is fully integrated with Structured’s semantic layer,
    meaning that all metrics and entities stored here are directly linked to the
    underlying data sources and transformations. This integration ensures
    consistency and accuracy across all reports, dashboards, and analyses.
  </Accordion>
  <Accordion title="Comprehensive Documentation" icon="file">
    Each entry in the Knowledge Base is accompanied by detailed documentation
    that explains the metric or entity, including its definition, calculation
    method, data sources, and any relevant business rules. This documentation
    helps ensure that everyone understands how metrics are derived and applied.
  </Accordion>
  <Accordion title="Searchable and Accessible" icon="magnifying-glass">
    The Knowledge Base is fully searchable, allowing users to quickly find the
    metrics, entities, or definitions they need. With user-friendly navigation
    and filtering options, it’s easy to locate specific information, whether
    you’re looking for a particular metric or trying to understand how a
    business entity is structured.
  </Accordion>
  <Accordion title="Version Control and History" icon="timeline">
    All changes to metrics and entities within the Knowledge Base are tracked
    through version control. This feature allows users to see the history of
    changes, understand why updates were made, and revert to previous versions
    if necessary. This ensures transparency and maintains the integrity of the
    data.
  </Accordion>
</AccordionGroup>

## **How to Utilize the Knowledge Base in Structured**

<Steps>
  <Step title="Step 1: Access the Knowledge Base" icon="book-open">
    Navigate to the Knowledge Base within Structured to access the centralized
    repository of metrics and entities. Use the search function or browse
    through categories to find the specific information you need.
  </Step>
  <Step title="Step 2: Explore Metrics and Entities" icon="chart-bar">
    Each metric or entity in the Knowledge Base includes detailed documentation.
    Review the definitions, calculation methods, and data sources to ensure that
    you understand how the metric is derived and how it should be used in your
    analyses.
  </Step>
  <Step title="Step 3: Leverage Integration with Semantic Layer" icon="plug">
    Use the integration with the semantic layer to see how the metrics and
    entities in the Knowledge Base are connected to your organization’s data
    sources. This connection ensures that all data used in reports and analyses
    is consistent and accurate.
  </Step>
  <Step title="Step 4: Track Changes with Version Control" icon="clock">
    If a metric or entity has been updated, review the version history to
    understand what changes were made and why. Use this information to ensure
    that your analyses reflect the most current and accurate data.
  </Step>
  <Step title="Step 5: Collaborate and Share Knowledge" icon="share-nodes">
    Share entries from the Knowledge Base with your team to ensure everyone is
    using the same definitions and metrics. Use the documentation and version
    history to support training and onboarding, helping new team members get up
    to speed quickly.
  </Step>
</Steps>
